SOC için ELK Stack Kurulumu

Elasticsearch, Logstash ve Kibana ile güvenlik operasyon merkezi kurma rehberi.

18 Temmuz 2025 20 dk okuma SOC Setup

ELK Stack (Elasticsearch, Logstash, Kibana), modern SOC operasyonlarının temel bileşenlerinden biridir. Bu rehber, SOC ortamında ELK Stack kurulumu ve konfigürasyonu için kapsamlı bir yaklaşım sunar.

ELK Stack Nedir?

ELK Stack, üç ana bileşenden oluşan açık kaynaklı bir log yönetimi ve analiz platformudur:

Elasticsearch

Dağıtılmış arama ve analitik motoru

Logstash

Log toplama ve işleme motoru

Kibana

Veri görselleştirme ve analiz arayüzü

Sistem Gereksinimleri

Minimum Sistem Gereksinimleri
  • CPU: 4 çekirdek (8 çekirdek önerilir)
  • RAM: 8GB (16GB önerilir)
  • Depolama: 100GB SSD (büyük SOC ortamları için daha fazla)
  • İşletim Sistemi: Ubuntu 20.04 LTS veya CentOS 8

Elasticsearch Kurulumu

1. Java Kurulumu

Elasticsearch Java gerektirdiği için öncelikle OpenJDK kurulumu yapalım:

sudo apt update
sudo apt install openjdk-11-jdk
java -version

2. Elasticsearch Repository Ekleme

wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | sudo apt-key add -
echo "deb https://artifacts.elastic.co/packages/7.x/apt stable main" | sudo tee /etc/apt/sources.list.d/elastic-7.x.list
sudo apt update

3. Elasticsearch Kurulumu ve Konfigürasyonu

sudo apt install elasticsearch
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch

Logstash Kurulumu

Logstash, çeşitli kaynaklardan log verilerini toplar, işler ve Elasticsearch'e gönderir.

1. Logstash Kurulumu

sudo apt install logstash
sudo systemctl enable logstash

2. Temel Logstash Konfigürasyonu

SOC ortamı için örnek Logstash konfigürasyonu:

input {
  beats {
    port => 5044
  }
  syslog {
    port => 514
  }
}

filter {
  if [fields][log_type] == "firewall" {
    grok {
      match => { "message" => "%{FIREWALLLOG}" }
    }
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "soc-logs-%{+YYYY.MM.dd}"
  }
}

Kibana Kurulumu

1. Kibana Kurulumu

sudo apt install kibana
sudo systemctl enable kibana
sudo systemctl start kibana

2. Kibana Konfigürasyonu

/etc/kibana/kibana.yml dosyasını düzenleyin:

server.port: 5601
server.host: "0.0.0.0"
elasticsearch.hosts: ["http://localhost:9200"]
server.name: "SOC-Kibana"

SOC için Dashboard Oluşturma

Temel SOC Dashboardları
  • Security Overview: Genel güvenlik durumu
  • Threat Intelligence: Tehdit istihbaratı göstergeleri
  • Network Traffic: Ağ trafiği analizi
  • Failed Logins: Başarısız giriş denemeleri
  • Malware Detection: Kötü amaçlı yazılım tespitleri

Güvenlik Konfigürasyonları

1. X-Pack Security Etkinleştirme

Elasticsearch güvenliğini artırmak için X-Pack Security özelliklerini etkinleştirin:

xpack.security.enabled: true
xpack.security.transport.ssl.enabled: true
xpack.security.http.ssl.enabled: true

2. Kullanıcı ve Rol Yönetimi

SOC operasyonları için farklı roller tanımlayın:

  • SOC Analyst: Log görüntüleme ve analiz yetkileri
  • SOC Manager: Dashboard oluşturma ve konfigürasyon yetkileri
  • Security Admin: Tam sistem yönetimi yetkileri

İzleme ve Bakım

Önemli Bakım Görevleri
  • Index lifecycle management (ILM) politikaları oluşturma
  • Düzenli yedekleme stratejisi uygulama
  • Sistem kaynaklarının izlenmesi
  • Log rotation ve retention politikaları
  • Güvenlik güncellemelerinin takibi

Sonuç

ELK Stack, SOC operasyonları için güçlü ve esnek bir platform sunar. Doğru konfigürasyon ve düzenli bakım ile organizasyonunuzun güvenlik izleme kabiliyetlerini önemli ölçüde artırabilirsiniz.

Bu kurulum rehberi temel bir başlangıç sağlar, ancak üretim ortamında ek güvenlik önlemleri ve performans optimizasyonları gerekebilir.
